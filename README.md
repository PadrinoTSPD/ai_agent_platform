# AI Agent Platform

简体中文说明文档 — 一个用于快速搭建和开发 AI Agent 平台的工程模板。

**目录结构**

- `backend/`: 后端服务（API、Agent 管理、模型接入等）
- `frontend/`: Web 前端（管理控制台、监控、调试界面）
- `mini-app/`: 小程序或轻量客户端示例

**项目简介**
这是一个用于构建 AI Agent 平台的参考实现/模板，包含后端、前端和小程序示例。

**主要功能**

- Agent 管理与生命周期（创建、启动、停止、监控）
- 多模型接入（本地模型 / 云模型适配层）
- 可扩展的插件与策略体系
- 前端控制台用于可视化管理与调试

详细部署步骤见部署脚本

**快速同步与提交bat使用方法**

### 前提条件
- 确保已正确配置 `.env` 文件（基于 `.env_temp` 模板创建）
- 本地环境已安装 Git 并配置好用户信息

### 可用脚本说明

#### 1. `push_to_my_remote.bat`
- **功能**：将本地修改提交并推送到个人远程仓库
- **使用方法**：双击运行，输入提交信息后自动执行 git add、commit 和 push

#### 2. `sync_local_with_my_remote.bat`
- **功能**：同步本地仓库与个人远程仓库的最新代码
- **使用方法**：双击运行，自动拉取远程仓库中对应分支的最新代码

#### 3. `sync_my_remote_with_upstream.bat`
- **功能**：将原主仓库的最新代码同步到个人远程仓库
- **使用方法**：双击运行，自动添加上游仓库、拉取最新代码、合并并推送到个人远程仓库

#### 4. `create_pr_to_upstream.bat`
- **功能**：打开浏览器创建 PR 到原主仓库
- **使用方法**：双击运行，自动在浏览器中打开 PR 创建页面

### 建议使用流程
1. 开始工作前：运行 `sync_my_remote_with_upstream.bat` 同步上游最新代码
2. 完成工作后：运行 `push_to_my_remote.bat` 提交到个人仓库
3. 准备贡献：运行 `create_pr_to_upstream.bat` 创建 PR
4. 定期同步：运行 `sync_local_with_my_remote.bat` 保持本地与个人远程同步

### 注意事项
- 首次使用前必须正确配置 `.env` 文件中的所有环境变量
- 合并冲突时需要手动解决，请仔细检查冲突文件
- 确保在运行脚本前没有未提交的重要修改
- 如有网络问题，请检查 Git 远程仓库配置和网络连接
